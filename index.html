Week 3 (Advanced CSS Layout)

We have came to the third week of Jeffrey's FED Expedition to CONQUER FED!!

This week started with ice-breakers again! During today's ice-breaker, I learnt that I have a classmate who experienced an EARTHQUAKE before?!?! Moreover, I also have a classmate who has 3 pet chickens!! This week's venture is a really long and technical one.

Today, we learnt that we could utilise the <form> Tag to create application forms. To allow users to interact with the forms, we can use input tags with types "text", "email", "password" and "submit". Furthermore, we can make add the placeholder attribute to give users context on what to fill in the input. Lastly, we can also make certain inputs compulsory by giving them the "required" attribute.

Next, we learnt about the selectors in CSS. There are multiple ways to select elements such as Descendant Selector and the Grouping Selector. When selecting elements in CSS, we are able to utilise the Specificity concept to select elements specifically.

CSS Box Model. This model is a concept that all define elements with Content, Padding, Border and Margin. To help us better understand this concept, Mr Henry got 2 of my fellow crewmates to stand beside each other said he wants them to have margin-right of 10% on the crewmate standing on the left. After this command, there was a wide gap between both of them!!

Moving on, we learnt about Media Queries. Media Queries are a powerful tool to change an element's CSS style based on certain criterias. With this, we are able to create responsive websites that can be viewed nicely on all devices

Here comes the most important part of our class. Flexboxes. Flexbox is a way to postion elements in an aesthetically pleasing manner. We can make an element into a flex box by adding the "display: flex" attribute. Then, to arrange the elements within a flexbox, we can utilise attributes such as "flex-direction" and "flex-wrap" and to align elements, we can use the "justify-content" and "align-items".

Week 5 (Javascript Basic)

Welcome to the fifth week of Jeffrey's FED Expedition to CONQUER FED!!

This week is rather interesting. This week is White Space Week, a week where no ICT students go to school for classes. Instead, we are supposed to learn exercise Asynchronous Learning and learn from home.

While it is unfortunate that there is no Ice-breakers this week, not having to wake up at 7am for class is really great!

The work we were tasked to complete this week is to complete a LinkedIn Learning course called "Learning the JavaScript Language". This course taught me about the various JavaScript's fundamentals. Furthermore, at the end of each chapter, there is a short quiz we have to complete which tests our understanding of each chapter.

The course started introducing us to the Variables and various Data Types. This chapter was a breeze, as it was similar to the variables and Data Types we know from Python and C#.

Moving on, we learnt about Objects, Arrays, Comments and Regular Expressions. Something new I learnt in this chapter was Regular Expressions. Regular Expression is a tool we can use to process strings and also helps ensure the input string by a user is in the correct format.

The next chapter was about Operators and Control Structures. One key highlight about this chapter is learning what Ternary Operators are. Ternary Operators follow the following syntax "condition ? expressionIfTrue : expressionIfFalse;". They a shorthand version of an if-else statement and is often used for simple, inline conditionals.

The following chapter was about Iteration with Loops. This chapter was also a breeze, as the concepts are similar to C#. However, a new topic i learnt was the Set and Map Data Types. Set is used to store each value exactly once, while Map is used to store key-value pairs whilst preserving the order of keys.

Then, we learnt about Functions in JavaScript. This chapter was slightly challenging as we learnt about new syntax to define functions, called an in-line function. This allows us to store functions in a variable which make the code more concise and readable.

Finally, we learnt about Asynchronous Functions. This type of function is extremely powerful, as it allows multiple functions to execute in a parallel manner, which improves a website's responsiveness.

After taking almost a day on this course, I was finally done! And as a token to recognise my effort, I was awarded a Certificate of Completion for this "Learning the JavaScript Language" Course, and I can now proudly display this certificate in my LinkedIn Profile!
